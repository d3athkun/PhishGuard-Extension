How It Works

Three steps from URL to verdict — happening in under a second.

01

Feature Extraction

90,021 features are extracted: character-level n-grams, word-level n-grams, and 21 hand-crafted URL signals like entropy, suspicious TLDs, and IP-in-host.

→
02

Model Inference

Your chosen model (or all three) classifies the feature vector as phishing or safe, returning a probability score for each class.

→
03

Verdict

Results appear instantly with a confidence percentage. Use Run All to compare all three models side-by-side.

The Models

All three models are trained on the same feature pipeline — dual TF-IDF + 21 URL features — so comparisons are fair.
